## 1. 安装 Python

### 步骤 1.1：下载 Python 安装包

1. 打开浏览器，访问 Python 官方网站：https://www.python.org/downloads/
2. 点击页面上的黄色大按钮 **"Download Python 3.x.x"**（会自动显示最新版本）
3. 下载完成后，您会得到一个类似 `python-3.12.x-amd64.exe` 的文件

> **💡 提示**：建议下载 Python 3.8 或更高版本

### 步骤 1.2：安装 Python

1. **双击**下载的安装包文件
2. **⚠️ 重要**：在安装界面的**底部**，勾选 **"Add Python to PATH"** 选项
   - 这一步非常重要！勾选后才能在命令行中使用 Python
3. 点击 **"Install Now"**（推荐安装）
4. 等待安装完成，看到 "Setup was successful" 提示后，点击 **"Close"**

![安装提示](https://docs.python.org/zh-cn/3/_images/win_installer.png)

## 2. 验证 Python 安装

安装完成后，我们需要验证 Python 是否正确安装。

### 步骤 2.1：打开命令提示符

1. 按键盘上的 **Win + R** 键
2. 在弹出的"运行"窗口中输入：`cmd`
3. 点击"确定"或按 **Enter** 键

### 步骤 2.2：检查 Python 版本

在命令提示符窗口中输入以下命令并按 **Enter**：

```bash
python --version
```

如果显示类似 `Python 3.12.0` 的版本号，说明安装成功！

### 步骤 2.3：检查 pip 工具

pip 是 Python 的包管理工具，用于安装第三方库。输入以下命令：

```bash
pip --version
```

如果显示类似 `pip 24.0 from ...` 的版本信息，说明 pip 也安装成功了！

> **❌ 如果出现错误**：请查看本文档末尾的 [常见问题解决](#6-常见问题解决) 部分

## 3. 下载项目代码

如果您已经有了项目文件夹（比如当前的 `ESP32S3_AUTO_BURN_TOOL`），请跳过此步骤。

### 如果是从 GitHub 下载：

1. 访问项目的 GitHub 页面
2. 点击绿色的 **"Code"** 按钮
3. 选择 **"Download ZIP"**
4. 下载完成后，右键解压到您想要的位置（比如 `J:\GitHub\`）

---

## 4. 安装项目依赖

### 步骤 4.1：进入项目目录

1. 打开命令提示符（方法见步骤 2.1）
2. 使用 `cd` 命令进入项目文件夹。例如，如果项目在 `J:\GitHub\ESP32S3_AUTO_BURN_TOOL`，输入：

```bash
cd /d J:\GitHub\ESP32S3_AUTO_BURN_TOOL
```

> **💡 提示**：
> - 如果路径有空格，需要用引号括起来，例如：`cd /d "J:\My Documents\ESP32S3_AUTO_BURN_TOOL"`
> - `/d` 参数允许切换到不同的驱动器（如从 C 盘切换到 J 盘）

### 步骤 4.2：安装依赖库

在项目目录下，输入以下命令：

```bash
pip install -r requirements.txt
```

这个命令会自动安装项目所需的所有依赖库：
- **esptool**：用于烧录 ESP32 芯片的工具
- **pyserial**：用于串口通信的库

安装过程可能需要几分钟，请耐心等待。看到类似 "Successfully installed ..." 的提示说明安装成功。

> **💡 国内用户加速技巧**：如果下载速度很慢，可以使用国内镜像源：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

## 5. 运行工具

本项目包含两个主要工具：

### 🔧 工具 1：ESP32 MAC 地址读取工具

用于批量读取 ESP32 芯片的 MAC 地址。

**运行命令**：
```bash
python esp32_readmac.py
```

### 🔥 工具 2：ESP32 固件烧录工具

用于给 ESP32 芯片烧录固件。

**运行命令**：
```bash
python esp32_flasher.py
```

## 📌 完整操作流程示例

假设您是第一次使用，下面是完整的操作步骤：

```bash
# 1. 打开命令提示符（Win + R，输入 cmd）

# 2. 验证 Python 安装

# 3. 进入项目目录

# 4. 安装依赖

```

## 6. 常见问题解决

### ❓ 问题 1：输入 `python` 命令后提示 "不是内部或外部命令"

**原因**：Python 没有添加到系统环境变量（PATH）

**解决方法**：
1. 重新运行 Python 安装程序
2. 选择 "Modify"（修改）
3. 点击 "Next"
4. 勾选 "Add Python to environment variables"
5. 点击 "Install" 完成修改
6. **关闭并重新打开命令提示符窗口**

### ❓ 问题 2：pip 安装依赖时速度很慢或超时

**解决方法**：使用国内镜像源

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

或者永久设置镜像源：
```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### ❓ 问题 3：提示 "权限不足" 或 "Permission denied"

**解决方法**：以管理员身份运行命令提示符

1. 点击开始菜单
2. 搜索 "cmd" 或 "命令提示符"
3. 右键点击，选择 **"以管理员身份运行"**
4. 重新执行安装命令

### ❓ 问题 4：运行工具时提示 "tkinter" 相关错误

**原因**：某些精简版 Python 没有包含 tkinter（图形界面库）

**解决方法**：
1. 重新安装 Python，使用官方完整版安装包
2. 在安装时选择 "Custom Installation"，确保勾选了 "tcl/tk and IDLE" 选项

### ❓ 问题 5：找不到串口或无法连接设备

**解决方法**：
1. 确保 ESP32 开发板已通过 USB 线连接到电脑
2. 检查设备管理器中是否能看到串口设备（COM 端口）
3. 如果没有串口，可能需要安装 CH340 或 CP2102 驱动程序：
   - CH340 驱动：https://www.wch.cn/downloads/CH341SER_EXE.html
   - CP2102 驱动：https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

### ❓ 问题 6：命令提示符中的中文显示乱码

**解决方法**：更改命令提示符编码

```bash
chcp 65001
```

这会将编码切换为 UTF-8，之后再运行工具。

## 📝 工具功能说明

### ESP32 MAC 地址读取工具 (esp32_readmac.py)

- **功能**：批量读取多个 ESP32 芯片的 MAC 地址
- **支持芯片**：ESP32 / ESP32-S2 / ESP32-S3 / ESP32-C3 / ESP32-C6 / ESP32-P4
- **特点**：
  - 自动检测串口
  - 自动识别芯片类型
  - 批量读取并保存
  - 自动去重
  - 支持导出到文本文件

### ESP32 固件烧录工具 (esp32_flasher.py)

- **功能**：给 ESP32 芯片烧录固件
- **支持芯片**：ESP32 / ESP32-S2 / ESP32-S3 / ESP32-C3 / ESP32-C6 / ESP32-P4
- **特点**：
  - 多串口并行烧录
  - 自动检测芯片型号
  - 支持多固件同时烧录（最多8个）
  - 自动保存配置
  - 支持自动烧录模式
  - 实时日志显示
  - 现代化图形界面
